Overview

Setup

bazel_dep(name = "rules_cc_autoconf", version = "{version}")

Autoconf-like configuration checks for Bazel.

This module provides a rule to mimic autoconf functionality, allowing you to generate config.h files by running compilation checks against a cc_toolchain.

autoconf

load("@rules_cc_autoconf//autoconf:defs.bzl", "autoconf")

autoconf(name, data, out, checks, package_name, package_version, template)

Generate configuration headers using autoconf-like checks.

This rule mimics autoconf functionality by running compilation checks against the configured cc_toolchain and generating a config.h file with the results.

Example:

autoconf(
    name = "config",
    out = "config.h",
    checks = [
        macros.AC_CHECK_HEADER("stdio.h"),
        macros.AC_CHECK_HEADER("stdlib.h"),
        macros.AC_CHECK_FUNC("printf"),
    ],
    package_name = "myproject",
    package_version = "1.0.0",
)

With file-based checks:

autoconf(
    name = "config",
    out = "config.h",
    checks = [
        macros.AC_TRY_COMPILE(
            define = "CUSTOM_CHECK",
            file = ":custom_test.c",
        ),
    ],
    data = [":custom_test.c"],
)

macros.AC_COMPUTE_INT

load("@rules_cc_autoconf//autoconf:defs.bzl", "macros")

macros.AC_COMPUTE_INT(define, expression, *, headers, language)

Compute an integer value at compile time.

WARNING: This macro is NOT cross-compile friendly. It requires compiling and running code to compute the value, which doesn't work when cross-compiling.

This matches GNU autoconf's AC_COMPUTE_INT signature: AC_COMPUTE_INT([VARIABLE], [EXPRESSION])

This is useful for computing values like sizeof(), alignof(), or other compile-time constants.

Example:

AC_COMPUTE_INT("SIZEOF_INT", "sizeof(int)")
AC_COMPUTE_INT("MAX_VALUE", "1 << 16")
AC_COMPUTE_INT("TWO", "1 + 1")

PARAMETERS

NameDescriptionDefault Value
defineDefine name for the result (first arg to match autoconf)none
expressionC expression that evaluates to an integer (second arg)none
headersOptional list of headers to includeNone
languageLanguage to use for check ("c" or "cpp")"c"

RETURNS

A JSON-encoded check string for use with the autoconf rule.

macros.AC_DEFINE(define, value)

Define a configuration macro unconditionally.

This is equivalent to GNU Autoconf's AC_DEFINE macro. It creates a define that will always be set in the generated config.h file.

Example:

AC_DEFINE("ENABLE_FEATURE", "1")
